## 1. 项目概述

## 1.1 产品一句话

**《云水江湖》是一款水墨国风武侠卡牌构筑 Roguelike 游戏，玩家操控貂蝉、蔡文姬、赵云、诸葛亮等角色，在被“墨灾”侵蚀的江湖中通过卡牌招式、心境变化、法宝心法和路线选择构筑自己的武学流派，最终面对心魔与改写命运的抉择。**

---

## 1.2 产品核心卖点

| 卖点 | 描述 |
|---|---|
| 水墨国风美术 | 以宣纸、水墨、朱砂、青绿山水构建差异化视觉 |
| 武侠卡牌战斗 | 每张牌都是一式武学、琴音、阵法或江湖手段 |
| 角色专属机制 | 四名角色拥有完全不同的资源和构筑逻辑 |
| 心境系统 | 宁、怒、悲、魅、乱、悟影响战斗、事件和结局 |
| 墨灾风险收益 | 强力墨灾牌带来高收益，同时积累墨痕与结局风险 |
| 招式链系统 | 根据出牌顺序触发武侠连招，增强出牌策略 |
| 江湖路线选择 | 普通战、精英、事件、商店、休息、Boss 组成单局探索 |
| 多结局叙事 | 根据玩家选择、心境和墨灾倾向导向不同结局 |

---

## 1.3 项目愿景

本项目不是简单的“国风版卡牌爬塔”，而是希望将：

```text
卡牌构筑 Roguelike
  +
武侠招式表达
  +
历史人物幻想
  +
水墨美术
  +
心境叙事
  +
高风险墨灾系统
```

融合为一款具有清晰文化识别度、强策略深度和可长期扩展内容空间的独立游戏。

最终希望玩家产生的感受是：

> **我不是在机械地打牌，而是在一局局江湖行旅中修炼属于自己的武学，面对角色无法逃避的命运。**

## 2. 市场与用户定位

## 2.1 目标用户

### 核心用户

| 用户类型 | 特征 |
|---|---|
| 卡牌 Roguelike 玩家 | 熟悉卡牌构筑、爬塔、随机奖励、路线选择 |

| 独立游戏玩家 | 接受非 3A 体量，重视机制创新和美术风格 |
| Steam 玩家 | 愿意购买 EA 或正式版独立策略游戏 |

### 次级用户

| 用户类型 | 特征 |
|---|---|
| 剧情向玩家 | 被角色故事、多结局、世界观吸引 |

| 想要叙事沉浸 | 角色个人线、心魔、章节剧情 |

---

## 2.3 产品竞争定位

### 竞品类型

| 类型 | 参考价值 |
|---|---|
| 卡牌构筑 Roguelike | 核心循环、奖励、路线设计 |
| 国风独立游戏 | 美术包装、文化表达 |
| 回合制策略游戏 | 敌人意图、状态、资源管理 |
| 剧情 Roguelike | 多局叙事、碎片化文本、结局回收 |

### 差异化方向

| 传统卡牌 Roguelike | 《云水江湖》差异 |
|---|---|
| 西方奇幻或抽象主题 | 水墨武侠、三国幻想、江湖奇遇 |
| 角色机制偏数值 | 角色机制强绑定人物气质 |
| 状态系统偏战斗 | 心境同时影响战斗、事件、结局 |
| 高风险牌通常只影响当前战斗 | 墨灾影响战后生命和结局倾向 |
| 出牌顺序多为局部策略 | 招式链将出牌顺序武侠化 |
| 剧情较弱或碎片化 | 主线章节和角色心魔明确 |

## 3. 产品目标

## 3.1 MVP 目标

MVP 的目标不是完成全部内容，而是验证核心体验是否成立。

### MVP 需要验证的问题

1. 回合制卡牌战斗是否足够好玩？
2. 角色专属机制是否有明显差异？
3. 水墨武侠包装是否提升体验辨识度？
4. 心境和墨灾是否能形成有意义的选择？
5. 玩家是否愿意重复开局尝试不同构筑？
6. 地图节点和奖励循环是否形成完整闭环？
7. 角色剧情是否能增强玩家投入感？

---

## 3.2 EA 初版目标

EA 初版面向 Steam 抢先体验或公开试玩后的持续开发。

目标：

- 完成 4 名可选角色。
- 完成 3 个主要章节和终章雏形。
- 构建 150+ 张卡牌内容。
- 构建 60+ 法宝。
- 构建 40+ 事件。
- 支持基础多结局。
- 提供稳定可反复游玩的单局体验。

---

## 3.3 正式版目标

正式版应具备：

- 完整四角色剧情线。
- 多结局完整回收。
- 更丰富事件与法宝。
- 更高难度模式。
- 图鉴、成就、每日挑战。
- 完整美术、音频、UI 和本地化。
- Steam 正式发售标准体验。

## 4. 核心玩法概述

## 4.1 单局核心循环

```text
选择角色
  ↓
进入章节地图
  ↓
选择路线节点
  ↓
战斗 / 事件 / 商店 / 休息 / 宝箱 / 精英 / Boss
  ↓
获得卡牌、金币、法宝、心法或承担代价
  ↓
优化牌组与构筑方向
  ↓
击败章节 Boss
  ↓
进入下一章
  ↓
面对心魔与最终 Boss
  ↓
通关或死亡结算
```

---

## 4.2 战斗核心循环

```text
回合开始
  ↓
触发回合开始效果
  ↓
抽牌
  ↓
获得能量
  ↓
查看敌人意图
  ↓
玩家出牌
  ↓
触发角色机制 / 招式链 / 法宝
  ↓
结束回合
  ↓
敌人行动
  ↓
进入下一回合
```

---

## 4.3 牌组循环

```text
抽牌堆 → 手牌 → 弃牌堆 → 洗牌回抽牌堆
```

特殊区域：

| 区域 | 说明 |
|---|---|
| 消耗堆 | 本场战斗移除，不再洗回 |
| 临时牌 | 战斗结束后移除 |
| 状态/诅咒 | 污染牌组，提供负面影响 |
| 保留牌 | 回合结束不弃置 |

## 6. 战斗系统需求

## 6.1 基础规则

### 需求描述

玩家和敌人在回合制框架下进行战斗。  
玩家通过抽牌、消耗能量、打出卡牌来造成伤害、防御、施加状态、触发角色机制。

### 功能要求

| 编号 | 需求 |
|---|---|
| BAT-001 | 每场战斗开始时，玩家牌组洗牌进入抽牌堆 |
| BAT-002 | 玩家每回合开始抽取默认 5 张牌 |
| BAT-003 | 玩家每回合获得默认 3 点能量 |
| BAT-004 | 玩家可使用能量打出手牌 |
| BAT-005 | 卡牌打出后进入弃牌堆或消耗堆 |
| BAT-006 | 回合结束时，未保留手牌进入弃牌堆 |
| BAT-007 | 抽牌堆为空时，弃牌堆洗回抽牌堆 |
| BAT-008 | 敌人根据意图在玩家回合后行动 |
| BAT-009 | 玩家生命归零判定战斗失败 |
| BAT-010 | 所有敌人生命归零判定战斗胜利 |

---

## 6.2 敌人意图

### 需求描述

敌人在玩家回合前展示下一次行动意图，玩家据此做出策略选择。

### 功能要求

| 编号 | 需求 |
|---|---|
| ENM-001 | 每个敌人每回合显示意图图标 |
| ENM-002 | 攻击意图显示预计伤害 |
| ENM-003 | 多段攻击显示次数与单段伤害 |
| ENM-004 | 防御、强化、弱化、召唤、蓄力等意图应有不同图标 |
| ENM-005 | 特殊 Boss 技能允许显示模糊或特殊意图 |
| ENM-006 | 玩家悬浮意图图标时，可查看详细说明 |

---

## 29. 风险与应对

## 29.1 玩法过度复杂

风险：  
心境、墨灾、角色资源、招式链同时存在，可能让新手难以上手。

应对：

- 默认推荐赵云。
- 第一章逐步解锁系统。
- 招式链可作为 P2，不在第一场战斗出现。
- 卡牌描述短句化。
- 悬浮提示完善。

---

## 29.2 角色机制失衡

风险：  
四名角色机制差异大，平衡成本高。

应对：

- 先做赵云和貂蝉验证上下限。
- 每名角色控制 2-3 个核心流派。
- 通用牌不提供过强专属替代。
- 建立胜率、平均战斗回合、Boss 击败率统计。

---

## 29.3 墨灾系统变成无脑强

风险：  
如果墨灾代价不足，玩家会无脑拿墨灾牌。

应对：

- 墨痕造成战后生命损失。
- 墨灾影响结局。
- 墨灾牌加入状态牌或诅咒。
- 强墨灾牌来源受限。
- 墨灾流需要专门法宝支撑。

---

## 29.4 心境系统存在感不足

风险：  
心境只停留在剧情，不影响战斗。

应对：

- 心境牌进入卡牌池。
- 心境影响事件选项。
- 法宝与心境联动。
- 终章回响读取心境。
- 每名角色至少有 1 条心境相关构筑。

---

## 29.5 美术成本过高

风险：  
水墨风对卡牌、角色、场景需求量大。

应对：

- MVP 阶段使用高质量关键图 + 复用背景。
- 卡牌插图可先用符号化水墨小图。
- 角色优先完成两名。
- Boss 和章节主视觉优先。
- 后续逐步替换占位资源。

---

## 29.6 剧情文本过多影响节奏

风险：  
大量剧情打断 Roguelike 节奏。

应对：

- 主线文本短句化。
- 事件文本控制篇幅。
- 允许快速跳过。
- 图鉴承载长文本。
- 战斗中只用短对白。

---

## 30. 验收标准

## 30.1 MVP 验收标准

MVP 达标应满足：

1. 玩家可以从选角色开始完成一局或死亡。
2. 战斗流程稳定，无关键阻断 Bug。
3. 玩家能理解能量、抽牌、敌人意图。
4. 至少 2 名角色玩法差异明显。
5. 至少 1 个章节地图可完整游玩。
6. 战斗胜利后能选牌并影响后续战斗。
7. 商店、休息点、Boss 节点可用。
8. 心境或墨灾至少一个特色系统可体验。
9. 玩家有“再开一局”的动力。
10. 美术方向能体现水墨武侠辨识度。

---

## 30.2 垂直切片验收标准

垂直切片应满足：

1. 第一章完整。
2. 赵云和貂蝉完整可玩。
3. 基础 UI 接近正式方向。
4. 至少一个 Boss 具备完整机制与演出。
5. 至少 15 个事件可随机出现。
6. 至少 20 件法宝可获得。
7. 卡牌升级、删牌、商店可用。
8. 水墨美术氛围明确。
9. 剧情主题能被玩家感知。
10. 测试玩家愿意重复游玩。

---

## 30.3 EA 版本验收标准

EA 版本应满足：

1. 四名角色全部可玩。
2. 三章流程完整。
3. 至少一个终章或简化最终 Boss。
4. 150+ 卡牌。
5. 60+ 法宝。
6. 50+ 事件。
7. 基础多结局。
8. 存档、设置、图鉴完整。
9. 崩溃率可控。
10. 达到 Steam EA 发布质量。

## 34. 总结

《云水江湖》的产品目标，是在成熟卡牌构筑 Roguelike 框架上，构建一个具有中国水墨审美、武侠招式表达和历史人物幻想的独特游戏体验。

它的核心设计公式是：

```text
易理解的卡牌战斗
  +
高重玩价值的构筑路线
  +
强差异化角色机制
  +
有代价的墨灾诱惑
  +
影响战斗与剧情的心境系统
  +
水墨武侠视觉包装
```

项目第一阶段应优先验证：

1. 战斗是否好玩；
2. 角色是否有差异；
3. 构筑是否有变化；
4. 墨灾和心境是否有存在感；
5. 美术是否有记忆点。

只要这五点成立，《云水江湖》就具备进一步扩展为完整商业独立游戏的基础。
